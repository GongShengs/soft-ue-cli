# DataAsset 参数详解文档模板

以下是生成文档的标准结构模板。实际输出时根据资产内容填充。

---

## 文档头部

```markdown
# {资产显示名} 参数详解

> **资产路径**：`{/Game/...完整路径}`
> **资产类型**：`U{类名}`
> **继承链**：`{子类} → {父类} → ... → DataAsset`
> **统计**：{N} 个属性 · {M} 个按钮
> **数据来源**：UE 编辑器 `inspect-details-panel` + AngelScript/C++ 源码分析

---
```

## 每个 Category 卷展栏

```markdown
## ▾ {Category 名称}

{1~2 句中文说明本卷展栏的功能用途，来自哪个父类}

### 按钮（如果有）

| 按钮 | 函数名 | 说明 |
|------|--------|------|
| **[{按钮文本}]** | `{函数名}` | {功能说明} |

### 属性表

| 属性 | 类型 | 控件 | 当前值 | 默认值 | 约束/元数据 | 说明 |
|------|------|------|--------|--------|------------|------|
| **{显示名}** | `{类型}` | {控件类型} | `{当前值}` | `{默认值}` | {ClampMin/Max/EditCondition 等} | {中文说明} |
```

## 枚举类型展开规范

当属性类型为枚举（enum）时，必须查找源码中的完整定义并列出所有值：

```markdown
#### {枚举名} 枚举完整定义

> 源码位置：`{文件路径}` 第 {行号} 行

| 枚举值 | 索引 | 说明 | 当前资产是否使用 |
|--------|------|------|-----------------|
| `{值1}` | 0 | {说明} | ✅ / ❌ |
| `{值2}` | 1 | {说明} | ✅ / ❌ |
| ... | ... | ... | ... |
```

## 结构体展开规范

当属性类型为结构体（struct）时，必须查找源码中的完整定义并列出所有字段：

```markdown
#### {结构体名} 结构体完整定义

> 源码位置：`{文件路径}` 第 {行号} 行

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| **{字段名}** | `{类型}` | `{默认值}` | {说明} |
```

## Map/Array 展开规范

当属性为 TMap 或 TArray 时，必须列出每个条目：

```markdown
#### {属性名} 条目详情（{N} 条）

##### ① {Key/索引} — {显示名或说明}

| 字段 | 当前值 | 说明 |
|------|--------|------|
| **{字段1}** | `{值}` | {说明} |

##### ② ...
```

## Transform 展开规范

FTransform 必须拆开为分量表格：

```markdown
| 分量 | X | Y | Z | W |
|------|---|---|---|---|
| **Rotation** | {X} | {Y} | {Z} | {W} |
| **Translation** | {X} | {Y} | {Z} | — |
| **Scale3D** | {X} | {Y} | {Z} | — |
```

如果值为 Identity 可标注 `> 当前值 = Identity（默认值）` 省略表格。
非 Identity 时必须列出完整数值。

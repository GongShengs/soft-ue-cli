# DataAsset 文档质量检查清单

生成文档后，必须逐项检查以下内容。任何不通过项必须修正后才能交付。

## 一、完整性检查

- [ ] 所有 Category 卷展栏均已列出（对照 inspect-details-panel 返回的 category 数量）
- [ ] 所有属性均已列出（对照 property_count）
- [ ] 所有按钮均已列出（对照 button_count）
- [ ] 每个属性都有「当前值」和「默认值」两列（即使相同也要写出）

## 二、类型展开检查

- [ ] **枚举类型**：每个枚举属性都已查找源码，列出枚举的**全部值**（不只是当前使用的）
  - 检查方法：`search_content` 搜索 `enum {枚举名}`
  - 必须标注哪些值被当前资产使用、哪些未使用
  - 必须标注源码位置（文件路径 + 行号）
  
- [ ] **结构体类型**：每个结构体属性都已查找源码，列出结构体的**全部字段**
  - 检查方法：`search_content` 搜索 `struct {结构体名}`
  - 必须标注哪些字段在当前资产中有值、哪些为默认
  - 必须标注源码位置（文件路径 + 行号）

- [ ] **TMap 类型**：每个 Map 条目都已逐个展开
  - Key 类型如果是枚举，必须列出枚举的全部值（含未使用的）
  - Value 中的子结构也要递归展开

- [ ] **TArray 类型**：每个数组元素都已逐个展开
  - 元素如果是结构体，按结构体规范展开

- [ ] **FTransform 类型**：每个 Transform 都已拆开为 Rotation/Translation/Scale3D 分量表
  - Identity 可简写标注，非 Identity 必须列出完整数值

## 三、代码分析检查

- [ ] 继承链中每个父类的源码位置已确认
- [ ] 属性的 `source_class` 已正确标注（来自哪个类）
- [ ] 代码注释中的 TODO/DEPRECATED 标记已在文档中体现
- [ ] 编辑器专用属性（EditorOnly）已标注 ⚠️
- [ ] EditCondition 依赖关系已说明（如「仅在 XX 勾选时可编辑」）
- [ ] 按钮关联的函数逻辑已简要分析（看源码中函数实现做了什么）

## 四、格式检查

- [ ] 表格列对齐，无断行
- [ ] 所有资产路径使用 `/Game/...` 格式
- [ ] 枚举值、类名、属性名使用反引号包裹
- [ ] 中文说明简洁清晰，每条不超过 2 句话
